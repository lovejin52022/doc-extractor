# 文档提取系统技术方案

参考项目：[Anything-Extract](https://github.com/ProgrammerAnthony/Anything-Extract)

## 一、整体架构

```
┌─────────────┐     ┌──────────────────┐     ┌──────────────┐
│   Frontend  │────▶│    FastAPI       │────▶│ QAnything    │
│ (Next.js)   │     │   Backend        │     │  OCR/PDF     │
└─────────────┘     └──────────────────┘     └──────────────┘
                              │
                              ▼
                        ┌──────────────┐
                        │    LanceDB   │
                        │ (向量数据库)  │
                        └──────────────┘
```

## 二、技术选型

### 后端栈
- **框架**: Python + FastAPI
- **AI 框架**: LangChain (高级 RAG 方案)
- **向量数据库**: LanceDB (默认，本地化部署)
- **LLM**: Ollama (phi3:mini, llama3.2:1b, qwen2:0.5b 等)
- **Embedding**: Ollama (nomic-embed-text, all-minilm 等)
- **OCR/PDF 解析**: QAnything OCR + PDF 模型服务

### 前端栈
- **框架**: Next.js + TypeScript
- **样式**: Tailwind CSS

## 三、核心功能模块

### 1. 文档上传与解析
- 支持格式：PDF, Word (.docx)
- 自动预处理：去噪、分页、文本提取
- 向量化处理：Embedding 模型生成向量表示

### 2. 标签配置系统
三种标签类型：
- **单选**: 从选项中选择唯一答案
- **多选**: 从多个选项中选择
- **填空**: 自由文本输入

示例配置:
```yaml
tags:
  - name: "作者"
    type: "single_select"
    options: ["张三", "李四", "王五"]
  - name: "主题"
    type: "multi_select"
    options: ["技术", "生活", "工作"]
  - name: "总结"
    type: "fill_blank"
```

### 3. 信息提取流程
1. 用户上传文档 → 2. OCR/PDF 解析服务处理 → 
3. 向量化并索引到 LanceDB → 4. 基于标签配置调用 LLM → 
5. 返回结构化结果

### 4. 高级检索方案
- Multi-Query Retrieval (多查询检索)
- HyDE (Hypernymer Decoding Embeddings)
- ParentDocumentRetriever (父文档检索器)
- RERANK (重排序)
- ES BM25 (搜索引擎)

## 五、部署方案

### 方式一：Docker (推荐)
```bash
# 一键启动全部服务
docker compose -f docker-compose-win.yaml --profile full up

# 仅启动后端 + 前端 (Ollama 在宿主机运行)
docker compose -f docker-compose-win.yaml up

# 停止服务
docker compose -f docker-compose-win.yaml down
```

### 方式二：本地安装
```bash
# 1. 创建虚拟环境并安装后端依赖
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt

# 2. 安装前端依赖
cd ../frontend
npm install

# 3. 启动服务
cd ..
start.bat  # Windows
```

## 六、模型配置 (.env)

```bash
# LLM 模型 (用于信息提取)
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=phi3:mini          # 推荐，平衡性能和速度

# Embedding 模型 (用于向量化)
OLLAMA_EMBEDDING_MODEL=nomic-embed-text

# 向量数据库路径
LANCE_DB_PATH=./storage/lancedb

# QAnything OCR/PDF 服务端口
QANYTHING_OCR_PORT=7001
QANYTHING_PDF_PORT=9009
```

## 七、核心 API 接口

### 文档管理
- `POST /api/upload` - 上传文档
- `GET /api/documents/{id}` - 获取文档详情
- `DELETE /api/documents/{id}` - 删除文档

### 信息提取
- `POST /api/extract` - 基于标签配置提取信息
- `GET /api/extract/{doc_id}` - 获取提取结果

### 检索查询
- `POST /api/search` - 多条件检索
- `GET /api/recommendations` - 推荐文档

## 八、性能优化建议

1. **模型选择**: 
   - CPU 部署：phi3:mini + nomic-embed-text (默认)
   - 极轻量：llama3.2:1b + all-minilm

2. **并发处理**: 使用 asyncio 处理多个文档并行解析

3. **缓存策略**: LanceDB 自动管理向量索引，支持快速检索

4. **资源监控**: 添加 Prometheus + Grafana 监控指标

## 九、扩展性设计

1. **LLM 替换**: 支持切换不同 Ollama 模型

4. **API 扩展**: FastAPI 设计便于添加新接口
